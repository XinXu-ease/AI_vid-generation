# 软件目录结构：

## 第一层是 本地数据层。
格式：JSON
### 第一类：固定资产库数据
比如 assets.json

里面存：

角色 id、名称、描述、参考图
服装列表
场景列表
道具列表
风格模板

示意结构可以是：

{
  "characters": [
    {
      "id": "char_a",
      "name": "主角A",
      "description": "20岁女生，黑长发，日系校园感",
      "looks": ["校服", "便装1"],
      "referenceImage": "/images/char_a.png"
    }
  ],
  "scenes": [
    {
      "id": "scene_cafe",
      "name": "咖啡馆内景",
      "style": "暖色电影感"
    }
  ],
  "props": [
    {
      "id": "prop_scarf",
      "name": "红色围巾"
    }
  ],
  "styles": [
    {
      "id": "style_jp",
      "name": "日系写实"
    }
  ]
}

### 第二类：短剧项目 / 剧集数据
比如 project.json

里面存：

剧集主题
故事梗概
分镜列表
每个分镜绑定的资产和节奏参数

### 第三类：生成结果 / 修订记录
比如 shots.json 或 history.json

里面存：

每个分镜当前版本
prompt
锁定项
是否参考上一镜
当前生成缩略图 / 假视频预览
修改历史

这样你的 demo 就会显得不是几张死页面，而是一个“有项目状态”的系统。

## 第二层是 前端交互层。
你这个项目最应该做的页面，不是很多，而是 3 个关键界面。

### 第一个：项目总览 / 剧集工作台
作用是让用户看到这一集有多少个分镜，整体的叙事阶段怎么分布。
建议包含：

剧集标题与故事简介
分镜时间线 / 卡片列表
每个分镜的阶段标签：铺垫、推进、转折、高潮、收束
每个分镜的状态：已配置、已生成、待修正

### 第二个：固定资产库页面或侧边栏
作用是管理角色、场景、道具、风格。
建议包含：

资产分类 tabs
资产卡片
缩略图 / 标签
“加入当前分镜”按钮
锁定等级选择

### 第三个：分镜结构化控制页面
这是核心。
建议包含 4 个区块：

左侧：分镜列表 / 时间线
中间：当前分镜配置表单
右侧：预览区 / 生成结果区
底部或右侧：版本历史 / 局部重生成按钮

当前分镜配置表单里至少要有：

叙事阶段
prompt 输入框
角色绑定
场景绑定
道具绑定
风格绑定
是否参考上一分镜尾帧
情绪强度滑条
节奏强度滑条
镜头运动单选
锁定等级设置

## 第三层是 伪 AI 交互层。
这一步很重要。即使你没有真正接模型，你也要让它“像一个 AI 产品”。

最简单的方法是做 3 个假逻辑：

第一，点击“生成分镜”后，右侧出现一个 loading 状态。
然后展示一个“生成结果卡片”，包括：

生成摘要
当前使用的资产
当前的节奏参数
一张假缩略图 / 占位视频
一个“系统解释”小框：说明 AI 是根据哪些条件生成的

第二，点击“局部重生成”后，只更新当前分镜卡片，不影响其他分镜。
这会非常贴合你的产品价值。

第三，做一个“一致性提醒”。
比如如果当前分镜没有选主角A，但上一镜选了主角A且强锁定，就弹出提示：
“检测到主角连续性可能中断，是否继续？”

这个功能很小，但会非常加分。因为它体现了你不是只做表单，而是做了“智能辅助”。

所以现在的路线可以概括成：

本地 JSON 管数据 + HTML/CSS/JS 做交互 + 假生成逻辑模拟 AI 工作流

# 示例项目 作为本地数据层的参考

准备一个完整的短剧示例，至少 4-5 个分镜。
比如：

分镜1：咖啡馆初遇，铺垫
分镜2：走出咖啡馆，推进
分镜3：雨夜争执，转折
分镜4：回忆闪回，高潮
分镜5：街头告别，收束

# 📌 项目执行计划：3 个可执行步骤

## 步骤 1：数据结构设计与本地数据层构建

**目标**：构建完整的本地数据模型，成为整个系统的基础。

**具体任务**：
- 设计并实现 assets.json（固定资产库）
  - 包含角色、服装、场景、道具、风格的完整定义
  - 每个资产配置 id、name、description、referenceImage
- 设计并实现 project.json（短剧项目数据）
  - 剧集主题、故事梗概
  - 分镜列表及每个分镜的资产绑定
- 设计并实现 shots.json（生成结果/修订记录）
  - 分镜版本管理、prompt 记录、锁定项
  - 生成缩略图路径、修改历史
- 准备完整示例项目数据
  - 创建 5 个分镜的完整数据（包含上述所有关键信息）
  - 验证数据间的关联与一致性

**产出物**：
- assets.json（固定资产库数据）
- project.json（剧集项目数据）
- shots.json（生成结果数据）
- 示例项目完整数据集

---

## 步骤 2：前端交互层开发

**目标**：实现 3 个关键用户界面，让用户能够可视化和管理数据。

**具体任务**：
- **界面1：项目总览 / 剧集工作台**
  - 显示剧集标题、故事简介
  - 分镜卡片或时间线视图
  - 每个分镜显示：阶段标签、当前状态、操作按钮
  - 支持点击进入某个分镜的详细编辑

- **界面2：固定资产库**
  - 资产分类 tabs（角色、场景、道具、风格）
  - 每个资产卡片展示缩略图、标签、描述
  - "加入当前分镜"按钮
  - 锁定等级选择（强锁定 / 优先保持 / 可变化）

- **界面3：分镜结构化控制页面（核心）**
  - 左侧：分镜列表或时间线导航
  - 中间：当前分镜配置表单
    - 叙事阶段单选
    - prompt 输入框
    - 角色 / 场景 / 道具 / 风格多选绑定
    - 是否参考上一分镜尾帧复选框
    - 情绪强度、节奏强度滑条
    - 镜头运动单选
    - 锁定等级设置
  - 右侧：预览区 / 生成结果区
  - 底部或右侧：版本历史和局部重生成按钮

**产出物**：
- HTML 页面结构（3 个主界面）
- CSS 样式表
- 基础 JavaScript 逻辑（数据加载、界面交互）

---

## 步骤 3：伪 AI 交互层与智能辅助功能

**目标**：让系统"像一个 AI 产品"，实现智能辅助和可信的用户体验。

**具体任务**：

- **分镜生成流程**
  - 点击"生成分镜"后，右侧出现 loading 状态
  - 显示生成结果卡片，包含：
    - 生成摘要
    - 当前使用的资产列表
    - 节奏参数摘要
    - 占位图 / 假缩略图
    - "系统解释"说明框（说明 AI 基于哪些条件生成）

- **局部重生成**
  - 点击"局部重生成"按钮后，只更新当前分镜卡片
  - 不影响其他分镜，维持整体一致性

- **一致性检测提醒**
  - 检测逻辑：当前分镜与上一分镜的元素连续性
    - 例如：如果上一镜选了主角A且强锁定，当前镜未选主角A，则弹提示
  - 提示框文案："检测到主角连续性可能中断，是否继续？"
  - 让用户感受到系统的"智能"

**产出物**：
- loading 和生成结果展示组件
- 局部重生成逻辑实现
- 一致性检测规则引擎
- 完整 demo 演示（可交互、有数据流动）

```
步骤1（数据） → 步骤2（交互） → 步骤3（AI层）
   ↓                ↓                ↓
必须先完成      依赖步骤1完成      依赖步骤2完成
```

**预期成果**：一个完整、可交互、具有智能提示的 AI 视频生成控制工作台原型。
